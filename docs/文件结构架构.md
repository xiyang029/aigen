# 文件结构架构

## 项目概览

`aigen` 是 Flutter/Dart 客户端应用，应用标题为“咕咕Do”。核心能力包括账号认证、AI 生图提交、任务列表与详情、图片发现页、提示词工具、API 配置管理、图片预览/保存和 Android APK 更新检查。

## 顶层目录

| 路径 | 作用 |
| --- | --- |
| `lib/` | Flutter 应用主代码，所有业务页面、模型、服务、共享组件在这里。 |
| `test/` | Flutter 测试，目前有应用启动冒烟测试。 |
| `assets/` | 应用图标等静态资源。 |
| `android/` | Android 平台工程、签名和 Gradle 配置。 |
| `ios/` | iOS 平台工程。 |
| `web/` | Web 平台入口与图标。 |
| `windows/`、`macos/`、`linux/` | 桌面平台工程。 |
| `build_arm64_release.bat` | Android arm64 release 构建脚本。 |
| `pubspec.yaml` | Flutter 依赖、版本号、资源和图标配置。 |
| `analysis_options.yaml` | Dart 静态分析规则，继承 `flutter_lints`。 |

## lib 目录架构

| 路径 | 作用 |
| --- | --- |
| `lib/main.dart` | 应用入口；初始化下载器；加载登录态；配置 `ShadApp.custom`、`MaterialApp`、底部导航和四个主 Tab。 |
| `lib/theme/app_theme.dart` | 全局主题、品牌色、间距常量、Dialog/Sheet/Button 主题。 |

| `lib/shared/system_toast.dart` | 系统提示封装，业务提示优先使用 `showAppToast`。 |

| `lib/shared/app_update.dart` | 应用更新检查、APK 下载后安装恢复、更新确认弹窗。 |
| `lib/services/api_client.dart` | 后端 Worker API 客户端；认证、生图任务、图库、配置、提示词工具、Release 更新、下载和缓存逻辑。 |
| `lib/services/auth_store.dart` | Token、Worker 地址、保存登录信息的本地持久化。 |
| `lib/models/image_task.dart` | 生图模式、任务、结果图、图库、分页、配置等数据模型。 |
| `lib/models/prompt_tools.dart` | 提示词 API 配置、图片反推和提示词改写结果模型。 |
| `lib/models/user.dart` | 用户和认证会话模型。 |
| `lib/utils/active_task_poller.dart` | 前台任务轮询，随生命周期暂停/恢复。 |

## 关键流程

### 启动与认证

`main()` 初始化 Flutter 与 `flutter_downloader`，`AigenApp` 调用 `ApiClient.load()` 读取本地 Token 和 Worker 地址。若存在 Token，会调用 `me()` 校验登录态；成功进入 `AppShellPage`，失败进入 `AuthPage`。

### 生图任务

首页 `ImageGenPage` 加载生图配置和进行中任务。提交时调用 `ApiClient.createImageTask()`；活动任务由 `ActiveTaskPoller` 定时刷新；点击任务进入 `ImageTaskDetailPage`，可复用提示词、参数和图片回填首页。

### 任务与图库

任务页调用 `fetchTasks()`，支持状态、模式、关键词和 cursor 分页。发现页调用 `fetchGallery()`，使用 `ImagePreviewCard` 和 `showImagePreviewOverlay()` 做预览，保存图片走 `saveNetworkImageToGallery()`。

### 提示词工具

`PromptToolsPage` 从 `fetchPromptConfigs()` 加载模型配置。图片反推先把本地图片转 data URL，再调用 `reversePromptFromImage()`；提示词改写调用 `modifyPrompt()`；结果可复制到剪贴板。

### 更新检查

`AppUpdateChecker` 通过 `ApiClient.fetchLatestRelease()` 查询 GitHub Release 中当前 ABI 的 APK，下载走 `downloadReleaseApkWithDownloader()`，并校验大小和 SHA-256。

## 查找与修改规则

- 修改前先按“任务到文件索引”定位入口，再用 `rg` 查调用链。
- UI 修改先找 `lib/shared/app_ui.dart` 是否已有通用组件；没有再在业务域内局部新增。
- 接口字段变更先改 `lib/models/`，再改 `lib/services/api_client.dart`，最后改页面展示。
- 新增后端接口方法放入 `ApiClient`，页面只消费清晰方法，不直接拼 HTTP。
- 新增跨页面能力放 `shared/` 或 `utils/`；只服务单一页面的组件放对应 `features/<业务域>/widgets/`。
- 平台目录只在平台能力相关需求中修改，避免为了普通 Flutter 页面改动原生工程。
